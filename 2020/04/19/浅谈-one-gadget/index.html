    什么是 one_gadget

    execve 原型：

    int execve(const char *pathname, char *const argv[],char *const envp[]);

    在 glibc 中，由于存在执行 /bin/sh 的相关需求，其中必定存在调用 execve 的代码片段。一般在比赛的 pwn 题目中，如果可以劫持程序流，并且有 leak libc 基址，就可以考虑直接劫持程序到 one_gadget 而不需要通过 system(“/bin/sh”) 的方式 getshell。

    手动寻找

    strings -a -tx /lib/x86_64-linux-gnu/libc.so.6 | grep /bin/sh

得到：18cd57 /bin/sh

objdump -d -M intel /lib/x86_64-linux-gnu/libc.so.6 | grep 18cd57 -A 30 -B 30|vi

    借助 vim 的搜索可以定位到第一处调用 execve 的地址 0x45294。由于是 64 位程序，传递进 execve 的前 3 个参数是 rdi，rsi，rdx。

    • rdi 代表第一个参数 const char *pathname ，也就是执行的文件，因为前面用 strings -a -tx /lib/x86_64-linux-gnu/libc.so.6 | grep /bin/sh 命令锁定了所有 /bin/sh 的地址，这里不会错。
    • rsi 代表第二个参数 char *const argv[]，用 execve 开 shell 的时候为 NULL 最好。（如果是执行其他程序，比如 ls 之类的，这里就需要是 ls 相应的 argv，因为 /bin/ls 会递归查找此数组的内容。）
    • rdx 代表第三个参数 char *const envp[]，一般也为空。

    要注意的细节就是，rsi 需要为 NULL。在实际的汇编中 rsi 并不一定为 NULL。
    从这里向上分别寻找 rdi，rsi，rdx 的最后一次赋值汇编：

       4526a:    48 8b 05 47 ec 37 00     mov    rax,QWORD PTR [rip+0x37ec47]        # 3c3eb8 <_IO_file_jumps@@GLIBC_2.2.5+0x7d8>
   45271:    48 8d 3d df 7a 14 00     lea    rdi,[rip+0x147adf]        # 18cd57 <_libc_intl_domainname@@GLIBC_2.2.5+0x197>
   45278:    48 8d 74 24 30           lea    rsi,[rsp+0x30]
   4527d:    c7 05 19 12 38 00 00     mov    DWORD PTR [rip+0x381219],0x0        # 3c64a0 <__abort_msg@@GLIBC_PRIVATE+0x8c0>
   45284:    00 00 00 
   45287:    c7 05 13 12 38 00 00     mov    DWORD PTR [rip+0x381213],0x0        # 3c64a4 <__abort_msg@@GLIBC_PRIVATE+0x8c4>
   4528e:    00 00 00 
   45291:    48 8b 10                 mov    rdx,QWORD PTR [rax]
   45294:    e8 d7 74 08 00           call   cc770 <execve@@GLIBC_2.2.5>

    上一条指令 mov rdx,QWORD PTR [rax]，把 rax dereference 后赋值给 rdx，所以需要向上找到 rax 的最后一次赋值，就是在 0x4526a。
    rdi 和 rsi 的最后一次赋值分别在 0x45271 和 0x45278。其中 rsi 的最后一次赋值为 lea rsi,[rsp+0x30] 所以为了正确开 shell，需要 [rsp+0x30] 为 NULL。这就是为什么 one_gadget 中会显示一些 constrains :

    0x45216 execve("/bin/sh", rsp+0x30, environ)
constraints:
  rax == NULL

0x4526a execve("/bin/sh", rsp+0x30, environ)
constraints:
  [rsp+0x30] == NULL

0xf02a4 execve("/bin/sh", rsp+0x50, environ)
constraints:
  [rsp+0x50] == NULL

0xf1147 execve("/bin/sh", rsp+0x70, environ)
constraints:
  [rsp+0x70] == NULL

    剩下的几个 one_gadget 的寻找方式和上面类似，也会因为需要 rsi 为 NULL 的原因，需要对应的 [rsp+xxx] 为 NULL。具体实现细节可以去看 one_gadget 作者的源码，其中用符号执行筛选相应的变量条件。
